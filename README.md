# EZ-EZ: Turn One Prompt Into Tested, Scored Website Variants

Give it a single idea (e.g. "a merch website for a coffee brand") and it generates
several real, live website variants, actually runs each one in an isolated sandbox,
scores them with a persona-style AI judge, and suggests concrete improvements
grounded in real current trend data.

## Overview

EZ-EZ takes a plain-language product idea and turns it into a small set of real,
working website variants - not mockups. Each variant is generated by an LLM, executed
live in its own cloud sandbox, and evaluated on a handful of real signals (does it
load, is it mobile-friendly, does a simulated persona find it usable). The best
variant is ranked to the top, and a separate step scrapes real current search trends
to ground a short list of improvement suggestions.

## How it works

1. **Describe your idea.** One prompt, e.g. "a merch website for a coffee brand."
2. **Generate variants.** The idea is expanded along 2 content/data concepts x 2
   visual styles = 4 variants. Each is a real, self-contained website generated by
   an LLM.
3. **Run them live.** Every variant is executed in its own isolated cloud sandbox and
   gets a real, clickable preview URL - not a screenshot or a static mock.
4. **Score them.** Each live variant is checked for basic quality signals (loads
   without errors, load time, key elements present, mobile-friendly) plus a
   persona-style review from an AI model acting as a shopper, and ranked.
5. **Suggest improvements.** Real, current search trends relevant to the idea are
   pulled and reasoned over to produce a short list of concrete, trend-grounded
   suggestions - not generic advice.

## Architecture

| Step | What it does |
|---|---|
| **Variant generation** | An LLM (OpenAI-compatible API) turns the prompt + a content/style variation into a real, self-contained website. |
| **Sandbox execution** | Each generated variant is written into its own isolated cloud sandbox, started for real, and exposed at a live, public preview URL. |
| **Trend scraping** | Real, current search-engine results relevant to the prompt are scraped to ground the suggestion step in actual data, not the model's guesses. |
| **Suggestion reasoning** | A second LLM call reasons over the scraped trend data to produce a short list of concrete, trend-grounded suggestions. |
| **Persona scoring** | A hosted evaluation model reviews each live variant like a real visitor would and gives a score and a one-line verdict. |
| **Ranking** | Persona scores are combined with a few deterministic checks (loads OK, load time, key elements present, mobile-friendly) into one ranked list with a clear winner. |

## Scope

This is a focused, working build:

- 2 content/data concepts x 2 visual styles = 4 variants per prompt
- 5 scoring signals: loads without errors, load time, key elements present,
  mobile-friendly, and one persona-style review
- One trend source per suggestion request
- A live preview URL per variant is the deliverable - no automated production
  deployment pipeline is wired up

## Tech stack

- **Frontend:** React + TypeScript + Vite, Tailwind CSS, Framer Motion, React Router
- **Backend:** Node.js + Express + TypeScript (`server/`), run via `tsx`
- Both processes run side by side locally; the frontend calls the backend through a
  `/api` proxy (see `vite.config.ts`)

## Getting started

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

Copy the example file and fill in real credentials:

```bash
cp .env.example .env
```

`.env` requires:
- `AIAND_API_KEY` - primary code-generation and suggestion-reasoning provider
- `DAYTONA_API_KEY` / `DAYTONA_API_URL` - sandbox execution
- `DOUBLEWORD_API_KEY` / `DOUBLEWORD_MODEL` - persona scoring
- `OXYLABS_USERNAME` / `OXYLABS_PASSWORD` - trend scraping

`MOONSHOT_*` variables are optional (an alternate code-generation provider, not
currently required for the app to run).

### 3. Run it

Start both the frontend and backend together:

```bash
npm run dev:all
```

Or run them separately in two terminals:

```bash
npm run dev        # Vite frontend on http://localhost:5173
npm run server:dev  # Express backend on http://localhost:8787
```

Open http://localhost:5173, describe an idea, and follow the flow through
generation, live previews, scoring, and suggestions.

## Project structure

```
src/            React frontend (pages, components, context)
server/         Express backend
  services/     One module per external integration (codegen, sandboxes,
                scraping, scoring)
  routes/       API endpoints consumed by the frontend
```
